# Family Fortunes Classroom Game

A browser-based classroom version of Family Fortunes designed for revision, recall, and keyword practice.

Created at King Edward VI School Southampton. Based on a PowerPoint resource from [craigndave.org](https://craigndave.org).

## What it does

- Lets teachers create and run Family Fortunes style classroom quiz boards.
- Supports two teams with editable team names and live score tracking.
- Uses survey-style rounds with one question and multiple hidden answers worth points.
- Works well for subject keyword revision, especially when boards are built around curriculum vocabulary.

## Current features

- Create, edit, reorder, and delete rounds in the setup screen.
- Save and load `.ffgame.txt` files.
- Built-in `games/` library shown inside the app through the **Games Library** button.
- Dedicated **How to Play** page linked from the setup screen.
- Theme picker with `Classic`, `Light`, `Sunset`, `Ocean`, and `Forest`.
- Optional theme music and bonus answers.
- Hidden question prompt in the live game with a **Reveal Question** button.
- Live reveal/hide controls for answers during play.
- Side team panels showing team name, round points, total points, strikes, and an `Add ❌` button.
- End game screen with winner display and confetti effect.
- Printable answer sheet download.

## Included game library

The bundled `games/` folder currently includes preset Computer Science packs for:

- GCSE Computer Science
- iGCSE Computer Science
- OCR A Level Computer Science

These files are indexed through `games/index.json` so they can be selected inside the app.

## Running the app

You can open the app directly in a browser for basic editing and local file loading, but the built-in `Games Library` works best when the app is served through a local web server because it uses `fetch()` to read files from `games/`.

### Simple local use

1. Download or clone the folder.
2. Make sure `index.html`, `app.js`, `styles.css`, and the audio files are in the same project folder.
3. Open `index.html` in a modern browser.

### Recommended local server

From the project folder, run one of these:

```bash
python3 -m http.server
```

Then open:

```text
http://localhost:8000
```

## Game file format

Game files are plain-text JSON saved as `.ffgame.txt`.

They now store only the revision content:

- `title`
- `rounds`
- each round's `question`
- each answer's `text`
- each answer's `points`

Gameplay state such as team scores, strikes, reveal state, theme choice, and bonus-answer selection is generated by the app at runtime and is not stored in the game files.

### Example

```json
{
  "title": "OCR A Level Computer Science - Software and Programming",
  "rounds": [
    {
      "question": "Name an OOP keyword",
      "answers": [
        { "text": "Class", "points": 24 },
        { "text": "Object", "points": 22 },
        { "text": "Encapsulation", "points": 20 }
      ]
    }
  ]
}
```

## Files of note

- `index.html`: main setup and live game page
- `how-to-play.html`: classroom instructions page
- `app.js`: game logic
- `styles.css`: styling and themes
- `games/`: bundled game files and `index.json`

## Notes

- The app currently assumes a two-team game.
- Bonus answers are visual and audio enhancements during play; they are not stored in the game files.
- If you build your own boards, keyword-based answers are usually the best fit for revision and exam recall.
